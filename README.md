# Что это?

Утилита для фильтрации содержимого файлов. Программа принимает на вход файлы с разными типами данных (Integer, Float, String), разбирает их и записывает в отдельные выходные файлы.

> Проект выполнен в рамках тестового задания для [ЦФТ ШИФТ (Java)](https://shiftproject.ru/courses/java/).

## Стек технологий
*   **Java 17** (Microsoft OpenJDK 17.0.16)
*   **Maven 3.9.11**
*   **Зависимости:** [Apache Commons CLI 1.6.0](https://commons.apache.org/proper/commons-cli/) — для парсинга аргументов командной строки.

## Установка и сборка

Можете создать проект в любой удобной среде разработки, но в целом достаточно использовать консоль (терминал).

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/qwerroaaa/file-content-filter-utility.git
    cd file-content-filter-utility
    ```

2.  **Соберите проект:**
    ```bash
    mvn clean package
    ```

3.  **Проект собран:**
    Если сборка прошла успешно, в папке `target/` появится исполняемый файл:
    `file-content-filter-utility-1.0-SNAPSHOT.jar`

## Инструкция по запуску

Запуск производится через консоль с помощью команды `java -jar`.

Доступные опции:
* -o <путь>	Указать путь для сохранения результатов (по умолчанию — текущая папка).
* -p <префикс>	Задать префикс для имен выходных файлов (например, result_).
* -a	Режим добавления: новые данные дописываются в конец существующих файлов (по умолчанию файлы перезаписываются).
* -s	Краткая статистика: выводит в консоль только количество записанных элементов.
* -f	Полная статистика: выводит мин/макс, сумму, среднее (для чисел) и размер строк.

**Базовый синтаксис:**
```bash
java -jar target/file-content-filter-utility-1.0-SNAPSHOT.jar [ОПЦИИ] <файл1> <файл2>...
```

Примеры запусков:
```bash
java -jar target/file-content-filter-utility-1.0-SNAPSHOT.jar in1.txt
java -jar target/file-content-filter-utility-1.0-SNAPSHOT.jar -f -p result_ -a in1.txt
java -jar target/file-content-filter-utility-1.0-SNAPSHOT.jar -o data/output -s in1.txt in2.txt
```
## Особенности
1. **Статистика:** при опции -a статистика собирается на основе исходных записей
2. Заполнение значений с E выглядит так: `1.123E10`, `1E10`, `1.4324E-12`
3. Ведущие нули считаются строкой, т.е. значения `00000000000000` считаюотся строкой, как и `0000000123`
4. Значения формата `5.` тоже считаются строкой. Правильно будет: `5.1`
5. Знаком разделителя считается `.`, а значит значение `1,123` считается строкой
